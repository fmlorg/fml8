fml5 のメール配送システムについて

fml4 と fml5 の相違点 fml5 の最大の目的の一つは、メンバーリストの取得と操作の統合と抽象化です。 配送には抽象化された MailingList クラスを次のように使います。

MailingList::* クラスの使い方について MailingList::* に属するクラスは SMTP および LMTP 配送へのインターフェ イスを提供します。 

MailingList::SMTP の実例

    use MailingList::SMTP;
    my $service = new MailingList::SMTP;
    if ($service->error) { Log($service->error); return;}

    $service->deliver(
                      {
                          mta             => '127.0.0.1:25',

                          smtp_sender     => 'rudo@nuinui.net',
                          recipient_maps  => $recipient_maps,
                          recipient_limit => 1000,

                          header          => $header_object,
                          body            => $body_object,
                      });
ここで $header_object はヘッダで、FML::Header オブジェクトです。 そして $body_oject はメール本文で、MailingList::Messages オブジェクトです。

コンポーネント C は C C C などへのインターフェイスです。
